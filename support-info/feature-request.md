# Internal Support Guide: Feature Request

This guide is for BeanBotics Inc. internal support staff handling feature requests for the Bean Machine.

## 1. Initial Actions
- Thank the customer for their feedback and interest.
- Review the request for clarity and completeness.

## 2. Information to Gather
- Customer name and email
- Product serial number (if relevant)
- Detailed description of the requested feature
- Use case or business need

## 3. Submission and Tracking
- Check if the feature is already on the product roadmap (if available)
- Submit new requests via the internal feature request process
- Inform the customer about the review process and expected timelines

## 4. Documentation and Follow-Up
- Log all details in the ticketing system
- Tag the ticket as "feature request" for product management review
- If the request is high-priority, recurring, or from a key customer, escalate to the product team for review

---
*All feature requests are reviewed quarterly. Encourage customers to provide as much detail as possible.*
